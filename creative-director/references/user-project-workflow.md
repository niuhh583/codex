## 总体判断

这是一个完整的视频广告制作项目库，不只是脚本文档。

项目资料覆盖：
- Brief / 需求
- 创意方案 / 策划方案 / Treatment
- 导演阐述
- 分镜脚本 / Shooting Board
- PPM / FPPM
- RUNDOWN / 通告单
- 报价 / 预算
- 时间表 / 排期
- 服装 / 美术 / 道具 / 场地
- 合成需求 / 三维调整 / 后期修改
- 剪辑工程 / AE 工程 / 素材 / 音效 / 成片

你的工作模式不是单点写脚本，而是：

`需求理解 -> 创意方向 -> 脚本分镜 -> PPM包装 -> 拍摄执行拆解 -> 后期合成管理 -> 修改交付`

默认项目推进链路：

1. `Brief / 需求表`：明确客户目标、产品卖点、交付规格。
2. `创意方案 / Treatment`：先讲调性、概念、故事或视觉机制。
3. `导演分镜 / Shooting Board`：把创意落成镜头。

6. `合成需求 / 三维调整`：把拍摄和后期衔接起来。
7. `修改意见 / 版本更新`：迭代到最终确认。

## PPM 的常见结构

从大量 PPM/PPT 文件看，你常用结构是：

```text
封面
CONTENTS / AGENDA
Director Treatment / 导演阐述
Shooting Board / 分镜脚本
Tone & Manner / 影调参考
Video Reference / 影片参考
Art Reference / 美术参考
Wardrobe Reference / 服装参考
Set Reference / 场景参考
Talent / 演员资料
Supporting / 制作协助
Schedule / 时间表
```

这意味着以后生成 PPM 文档时，不能只给分镜表，要补齐客户提案可理解的“包装逻辑”：
- 为什么这么拍。
- 影调是什么。
- 画面参考是什么。
- 美术/服装/场景怎么落地。
- 哪些部分实拍，哪些部分 AI/CG/三维/合成。

## 分镜脚本习惯

你的导演分镜有两个常见形态：

### 1. Word 表格型

```text
镜号（shot）
分镜（STB）
画面内容（Description）
备注（Dialouge）
```

画面内容写法：

```text
景别/角度/运镜 + 主体动作 + 产品/剧情信息
```

备注写：

```text
VO / 字幕 / 台词 / 卖点 / 实拍 / AI / CG / 合成 / 道具 / 拍摄提醒
```

### 2. PPT 版分镜型

常见于客户提案或 PPM：

```text
镜号 + 景别/角度/运镜
画面描述
备注 / VO / Super / 实拍+CG
参考图或分镜图
```

PPT 分镜比 Word 更重视：
- 视觉参考。
- 影调统一。
- 客户可读性。
- 每页信息不要太密。

## RUNDOWN / 通告思维

RUNDOWN 文件反复出现，结构稳定：

```text
项目名 拍摄-RUNDOWN
Producter / 监制
Director / 导演
DP / 摄影指导
Arts Direct / 美术指导
Producer / 制片
Shooting Day / 拍摄日
Breakfast / Lunch / Dinner
Talents Call / 演员通告
拍摄日执行安排
时间
画面内容 Content描述
VO
备注
场地 / 艺人组 / 产品组
```

重要原则：
- RUNDOWN 是从 final 脚本和 final PPM 反推出来的。
- RUNDOWN 不是创意文档，而是现场执行工具。
- 分镜里的镜头要能被拆成拍摄顺序、场地顺序和组别顺序。
- 常按场地、艺人、产品、夜景、CG 配合来重排，而不是按影片时间顺序拍。

以后如果输出拍摄执行稿，要默认考虑：
- 哪些镜头同场地合并。
- 哪些需要先拍产品空镜。
- 哪些是艺人组。
- 哪些是产品组。
- 哪些镜头需要 CG/穿梭机/伸缩炮/摇臂/三维配合。
- 哪些是高风险镜头，需要保底方案。

```text

项目名｜内部版_V1.docx
```

## 创意和执行的关系

项目库体现出一个很强的工作习惯：

`创意要能进入 PPM，PPM 要能进入 RUNDOWN，RUNDOWN 要能进入现场，现场要能进入后期。`

因此后续写任何脚本都要避免只停留在文学表达。

每个重要镜头最好同时考虑：
- 客户为什么会买这个创意。
- 现场怎么拍。
- 美术/道具/服装需要准备什么。
- 是否需要 AI/CG/三维/合成。
- 后期是否有明确素材输入。
- 是否能在预算和时间内完成。

## 实拍 + AI / CG / 三维工作方式

从文件中反复出现：

```text
实拍+CG
实拍+AI合成
AI制作
AIGC
三维
合成需求
三维调整建议
```

默认判断：
- 人物表演、产品操作、真实质感尽量实拍。
- 大场景、幻想世界、转场、能量线、宝箱、异世界、历史宏观叙事可用 AI/CG。
- 产品结构、数据证明、功能点需要清楚，不可被特效淹没。
- 合成镜头要在分镜阶段就标注，不能拍完再临时想。

## 后续我应采用的工作模式

当用户给一个新项目时，我应按以下顺序工作：

1. 读取 Brief / 卖点 / 素材 / 参考。
2. 提炼核心商业目标和受众。
3. 判断片型：剧情广告、卖点展示、教程安装、游戏宣传、品牌片、纪录片、TVC。
4. 给 1-3 个创意方向，说明推荐方向。
5. 用户确认方向后，进入脚本。
6. 按用户既有格式输出导演分镜。
7. 如用户说“输出文档”，生成 `.docx`。
8. 如进入提报，补 PPM 结构：导演阐述、影调、参考、美术、服装、执行。
9. 如进入拍摄，生成 RUNDOWN / 通告 / 镜头执行拆解。
10. 如进入后期，生成合成需求和修改意见清单。
