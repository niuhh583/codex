## 核心片型

### 1. 剧情广告型

适合家居、母婴、厨房、宠物、消费电子等有生活情绪的产品。

结构：
1. 用生活混乱或尴尬开场。
2. 人物产生反应，形成轻喜剧或小冲突。
3. 产品以转场、降临、变形、替换或状态切换的方式登场。
4. 产品卖点逐个解决前面的痛点。
5. 结尾回到生活变好的状态，落产品定版。

代表：
- 水槽：BBQ 洗不了 -> 小盆不够 -> 水槽像拯救者登场。
- 气动床：房间乱 -> 收纳崩溃 -> 气动床改变空间。
- 云朵沙发广告：情侣选择冲突 -> 沙发颜色/套件变化 -> 不必二选一。
- 大画家：孩子乱画 -> 学习机把想象力导入数字画布。

### 2. 卖点展示型

适合功能明确、结构明确、客户重点是参数和性能的产品。

结构：
1. 产品 Hero 或场景建立。
2. 按最重要卖点排序。
3. 每个卖点对应一个动作证据。
4. 特写补结构，字幕补参数。
5. 生活场景或多场景收尾。

代表：
- 金属梯：推到泳池边 -> 宽踏板 -> 加强结构 -> 承重 -> 扶杆 -> 螺丝 -> 定版。
- 泳池台阶：雨滴质感 -> 安装 -> 1000LBS -> 防滑 -> 储物防水 -> 多场景。
- 油烟机卖点：外观 -> 油烟 -> 开机 -> 吸力 -> 集油杯 -> 清洁 -> 收纳 -> 尺寸。

### 3. 教程安装型

适合安装步骤复杂、用户需要跟做的产品。

结构：
1. 先给使用场景和产品结果。
2. 展示配件。
3. 按步骤逐条拆解。
4. 每步只拍关键动作，避免冗余。
5. 完成后展示使用状态和极端环境。

代表：
- 车衣安装视频：房车场景 -> 配件 -> 排水槽罩 -> 梯子罩 -> 铺设 -> 固定 -> 防风带 -> 轮胎罩 -> 修补 -> 恶劣天气。

## 卖点转化原则

不要把卖点停留在文字里。每个卖点都要变成动作、结果或物理证据。

- 承重：人踩、重物落下、产品不晃。
- 稳固：周围震动，主体不动。
- 容量：真实物品放进去，俯拍看满。
- 防水：雨滴、水流、打开后内部干燥。
- 防滑：湿脚踩下，纹理特写，动作稳定。
- 易清洁：冲洗、擦拭、磁吸装回。
- 可调节：人物实际调高度或角度。
- 可收纳：物品放入、合上、空间变整洁。
- 强结构：加强筋、螺栓、支撑杆、三角结构、金属纹理。
- 低噪：人物能正常看视频、吃饭、交流。
- 便携：拿起、移动、放进柜子。

## 镜头语言模板

常用写法：

`景别 / 角度 / 运镜 + 主体动作 + 产品信息 + 备注`

高频镜头：
- `全景/侧拍/推进`：建立场景、产品位置。
- `中景/侧拍/手持`：人物反应、生活感、轻喜剧。
- `特写/侧拍/固定`：按键、材质、结构、卡扣。
- `特写/俯拍/推进`：容量、桌面混乱、操作步骤。
- `全景/俯拍/固定`：收纳空间、整体布局。
- `环绕/拉出`：产品高级感和结构完整性。
- `一组镜头`：快速展示多个功能、安装步骤或多场景。
- `甩镜/爆白/遮挡转场`：剧情变化、产品切换、AI 合成切点。
- `升格`：水滴、材质、稳定性、冲击瞬间。

## AI / CG / 实拍边界

文档里的常见习惯：
- 场景大环境可用 AI：山林、道路、异世界、恶劣天气、城市大景。
- 产品关键卖点尽量实拍：材质、结构、操作、承重、安装。
- 夸张登场可用合成：宝箱降临、产品变形、沙发换色、游戏世界转换。
- 后期标注必须服务理解：尺寸、数据、卖点英文字幕、轮廓线。
- AI 画面要有物理逻辑：不能悬浮、要有抖动后稳定、物体位置不能挡 LOGO。

## 修改意见反推的审美标准

从 `修改意见.docx` 看，修改重点集中在：
- 物体不能无理由悬浮，必须有真实接触或受力逻辑。
- 关键主体需要居中或更明显。
- 道具不能挡住 LOGO。
- 动画变形要有过程：抖动、稳定、掉落、填充等。
- 关键动作要更快、更清楚。
- 小物件也要有收起、松开、落下等动作闭环。

核心标准：

`可以夸张，但不能没有物理逻辑；可以炫，但必须看懂产品和卖点。`

## 后续脚本默认执行准则

1. 先问或判断片型：剧情广告、卖点展示、教程安装、游戏宣传、故事概要。
2. 先提炼核心矛盾或核心测试，不直接堆镜头。
3. 把最强卖点放在高潮或开场钩子。
4. 其他卖点按证据链顺序排列，解释为什么核心卖点成立。
5. 每个卖点都要有一个可拍动作。
6. 每个镜头都写清楚景别、角度、运镜、动作。
7. 备注中明确字幕、旁白、实拍、AI、CG、后期标注。
8. 控制预算时优先保留核心证据镜头，删掉氛围镜头。
9. 结尾必须有产品 Hero、主卖点、LOGO 或购买理由。
10. 所有炫酷表达都必须服务商业逻辑。

11. 写脚本时同步思考后续 PPM、RUNDOWN 和后期合成需求，避免只写文学化画面。
12. 重要镜头要标清实拍、AI、CG、三维、合成和现场执行难点。
13. 如果用户进入项目执行阶段，要能继续输出 PPM资料、拍摄RUNDOWN、合成需求、修改意见和时间表。

## 默认输出文档格式

以后输出导演分镜文档时，默认使用用户既有格式，不另起复杂版式。

### 文档头

```text
项目名｜导演分镜
DIR_SHOOTINGBOARD_V1
Date: YYYY/MM/DD
Film Time: 30s / 40-50s / 60s / 60-120s
```

如果需要，可在表格前加一小段创意说明，但不要太长。说明通常包括：
- 影片调性
- 核心创意或核心测试
- 实拍 / AI / CG / 后期边界

### 主表格

固定四列：

| 镜号（shot） | 分镜（STB） | 画面内容（Description） | 备注（Dialouge） |
|---|---|---|---|

列使用规则：
- `镜号（shot）`：用 `1`、`2`、`3` 或 `1-1`、`1-2` 表示镜头和子镜头。
- `分镜（STB）`：通常留空，后续可放图或分镜缩略图。
- `画面内容（Description）`：必须写成 `景别/角度/运镜 + 动作 + 产品信息`。
- `备注（Dialouge）`：放字幕、旁白、台词、卖点、实拍/AI/CG、道具或拍摄提醒。

### 场景段落

场景单独成行，放在镜头组之前：

```text
户外庭院-日
厨房-日
电竞房-夜
日-白墙-卖点一
SKU1&开场&山间马路
```

## 应用于《6000LB 重力审判》

这支货架片应该归为“卖点展示型 + 测试事件型”。

推荐结构：

`暗场产品亮相 -> 重量测试启动 -> 承重逐级升级 -> 结构证据链 -> 调节/锁定等辅助卖点 -> 6000LBS 高潮 -> 产品定版`

- 不真实堆满 6000LBS，采用安全配重、仿真重物和后期合成。
- 画面可以工业硬核，但必须像真实可拍广告，不要变成纯 CG 概念片。
